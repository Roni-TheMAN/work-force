# Work Hours Design System (Strict)

## Purpose

This file defines the design language for the Work Hours project.

All UI generated by Codex must follow these rules.
These rules override default styling decisions.

---

## Core Design Language

The UI must feel:

- professional
- polished
- minimal
- modern SaaS
- smooth, not flashy
- consistent across all pages

References in spirit only:
- ShadCN
- Linear
- Stripe
- ElevenLabs

Do not copy any brand directly.

---

## Non-Negotiable Rules

1. Use ShadCN components whenever possible.
2. Use CSS variables for all core colors.
3. Never hardcode Tailwind palette colors like `bg-blue-500`, `text-red-600`, etc.
4. Use design tokens only:
    - `bg-background`
    - `text-foreground`
    - `bg-card`
    - `text-card-foreground`
    - `border-border`
    - `bg-primary`
    - `text-primary-foreground`
    - `bg-secondary`
    - `text-secondary-foreground`
    - `bg-muted`
    - `text-muted-foreground`
    - `bg-accent`
    - `text-accent-foreground`
    - `ring-ring`
5. Border radius must be consistent.
6. Motion must be subtle and fast.
7. Layout must be structured and breathable.
8. All screens must feel like one product, not mixed styles.

---

## Color System

All theme values must come from CSS variables.

Allowed approach:
- HSL variables in `theme.css`
- Tailwind utilities mapped to CSS variables
- optional `data-theme` switching for multiple brand themes

Forbidden:
- random hex values inside components
- inline colors
- arbitrary Tailwind color usage

---

## Typography

Use one font family consistently.

Recommended:
- Inter
- system sans fallback

Allowed text hierarchy:
- page title: `text-2xl font-semibold tracking-tight`
- section title: `text-lg font-semibold`
- card title: `text-base font-semibold`
- body: `text-sm`
- helper text: `text-sm text-muted-foreground`
- labels: `text-sm font-medium`

Do not use too many sizes.

---

## Surfaces

Page background:
- `bg-background`

Grouped content:
- `bg-card border border-border rounded-2xl`

Muted sections:
- `bg-muted`

Avoid shadow-heavy UI.
Prefer borders over heavy shadows.

---

## Interaction

Hover:
- subtle background shift
- slight scale only when appropriate

Focus:
- visible ring using `ring-ring`
- accessible focus states required

Active:
- clear but subtle

Disabled:
- lower opacity
- no confusing hover states

---

## Responsiveness

UI must work well on:
- mobile
- tablet
- desktop

Rules:
- stack content naturally on smaller screens
- avoid cramped side-by-side layouts on mobile
- preserve spacing rhythm across breakpoints

---

## Accessibility

- sufficient contrast required
- focus states required
- clickable targets should be large enough
- forms must have labels
- status colors cannot be the only signal

---

## Forbidden Patterns

Do not generate:
- random gradients everywhere
- glowing gaming-style UI
- inconsistent radius
- inconsistent spacing
- mixed visual languages
- long flashy animations
- deeply nested wrappers without reason
- multiple competing accent colors

---

## Codex Enforcement

Before generating UI, Codex must:
1. read `/design/design.md`
2. read `/design/components.md`
3. read `/design/layout.md`
4. read `/design/motion.md`

Then Codex must generate UI that follows these rules strictly.

If a request conflicts with this design system, follow the design system.
